Grafos estão em todo lugar

  • Moléculas (átomos + ligações)
  • Redes sociais (pessoas + conexões)
  • Circuitos (componentes + fios)
  • Grafos de conhecimento (entidades + relações)
Objetivo: aprender a gerar novos grafos a partir de dados observados — descoberta de fármacos, design de materiais, modelagem de redes.

O problema central: invariância a permutação

Um grafo com \(n\) nós admite até \(n!\) permutações de rótulos — todas representam o mesmo grafo.
  • Redes neurais operam sobre tensores e sequências ordenadas.
  • Para treinar um modelo, precisamos fixar uma ordem dos nós.
  • Tensão fundamental: estrutura invariante vs. representação ordenada.

De grafo para sequência de ações

Dado um grafo \(G = (V, E)\) e uma ordenação \(\sigma = (v_1, \ldots, v_n)\):

AddNode → AddEdge → ChooseDest → StopEdge → … → StopNode

Ordenação A: \(\sigma_A = [1,2,3,4,5]\)

[AddNode, StopEdge, AddNode, AddEdge, ChooseDest(1), StopEdge, …]

Ordenação B: \(\sigma_B = [3,5,1,4,2]\)

[AddNode, StopEdge, AddNode, AddEdge, ChooseDest(1), StopEdge, …]

Mesmos tokens, destinos diferentes!

Mudar a ordenação dos nós muda toda a sequência de decisões que o modelo vê durante o treino.

Geradores autorregressivos de grafos

Modelos que constroem grafos passo a passo, por decisões locais condicionadas no histórico de construção.
  • A cada passo, o modelo decide: adicionar nó? adicionar aresta? para qual destino?
  • O treino usa teacher forcing sobre a sequência supervisionada.
  • Diversas abordagens na literatura (GraphRNN, GRAN, etc.).
  • Neste trabalho, usamos o DGMG como ambiente controlado.
Pergunta central
"A ordenação dos nós é um detalhe de implementação
ou uma escolha de modelagem com impacto real?"

Tese: a ordenação é uma variável de modelagem — e pode ser otimizada.

Interpretação: ordenação molda a entropia das decisões

  • No passo ChooseDest, o modelo seleciona entre nós já inseridos.
  • Se o destino correto é "óbvio" no prefixo, a decisão é fácil (baixa entropia).
  • Se muitos nós anteriores são plausíveis, a distribuição fica difusa (alta entropia).
Localidade: boa ordenação tende a colocar vizinhos do nó atual próximos na sequência, reduzindo a ambiguidade efetiva das decisões.

A "qualidade" de uma ordenação depende da topologia — não há regra universal.

Conclusão do bloco

"Nenhuma heurística fixa é universalmente ótima.
E se a ordenação fosse aprendida?"

Motivação para a contribuição principal: usar Reinforcement Learning para aprender a ordenação.

Bloco 3 · Contribuição principal

Formulação bilevel

Nível superior:
Política \(\pi_\theta(\sigma \mid G)\)
Escolhe a ordenação dos nós
Nível inferior:
Gerador DGMG \(p_\phi\)
Treina com a sequência resultante
\[ \phi^\star(\theta) = \arg\min_\phi\; \mathbb{E}_{G \sim \mathcal{D}}\; \mathbb{E}_{\sigma \sim \pi_\theta(\cdot \mid G)} \left[\mathcal{L}_{\text{NLL}}(\phi, \mathcal{T}(G, \sigma))\right] \]

A política quer maximizar a qualidade gerativa do gerador treinado.

O Processo de Decisão de Markov

  • Estado: \(s_t = (G, S_t)\), onde \(S_t \subseteq V\) é o conjunto de nós já selecionados.
  • Ações: escolher próximo nó em \(V \setminus S_t\).
  • Transição: \(S_{t+1} = S_t \cup \{u_t\}\).
  • Episódio: termina quando \(|S_t| = n\) (permutação completa).
  • Recompensa: atrasada — atribuída após treinar e avaliar o gerador.
\[ \pi_\theta(\sigma \mid G) = \prod_{t=1}^{n} \pi_\theta(u_t \mid G, S_t) \]

É um MDP condicionado ao grafo: cada grafo gera um episódio diferente.

Arquitetura da política

GNN encoder → Pooling global → MLP score → Softmax mascarado
  • GNN codifica cada nó usando estrutura do grafo.
  • Pooling agrega informação global para contexto.
  • MLP pontua cada nó candidato (não selecionado).
  • Softmax mascarado garante que nós já selecionados recebem probabilidade zero.
Indicador de nó já selecionado concatenado ao embedding — permite que a GNN distinga nós disponíveis de nós já ordenados.

A recompensa: por que ela é composta?

\[ R(\tau) = R_{\text{pred}}(\tau) + \lambda\, R_{\text{struct}}(\tau) \]
\(R_{\text{pred}}\)
NLL de validação do DGMG treinado.
Mede o quanto o gerador aprendeu a distribuição.
\(R_{\text{struct}}\)
Métricas estruturais das amostras geradas.
Tamanho válido, conectividade, Gini, hubiness.
Por que dois componentes? NLL baixo não garante grafos estruturalmente corretos — o modelo pode gerar grafos com boa likelihood mas topologia errada.

Gradiente de política (Teorema 6.1)

\[ \nabla_\theta J(\theta) = \mathbb{E}_{G \sim \mathcal{D}}\; \mathbb{E}_{\sigma \sim \pi_\theta(\cdot \mid G)} \left[(R(\tau) - b)\, \nabla_\theta \log \pi_\theta(\sigma \mid G)\right] \]
  • REINFORCE: estimador de gradiente via amostragem de trajetórias.
  • A ordenação é discreta — backprop não passa pela seleção dos nós.
  • A recompensa é atrasada — só existe após treino e avaliação do gerador.
  • \(b\) é um baseline para redução de variância.

Policy gradient é a ferramenta natural: decisão discreta + reward não-diferenciável.

Otimização alternada (Algoritmo 2)

Política propõe \(\sigma\) → Tradutor gera sequência → DGMG treina → Avaliação → Update política
↺ loop iterativo
  • A cada iteração externa, a política e o gerador se co-adaptam.
  • O gerador melhora com as novas ordenações; a política melhora com o feedback do gerador.
  • Necessário porque a qualidade de uma ordenação depende do estado atual do gerador.

Attention RL Ordering: três inovações

1. Encoder com atenção

Mecanismo de atenção tipo transformer permite capturar dependências de longo alcance entre nós, além de vizinhos imediatos.

2. Três camadas de message-passing

Amplia o campo receptivo: cada nó "vê" vizinhos a 3 saltos, capturando estrutura mesoscópica.

3. Warm start por grau

Primeira iteração usa ordenação por grau decrescente como prior — acelera convergência e estabiliza treino inicial.

Por que isso é diferente de uma heurística fixa?

  • Heurística fixa aplica sempre a mesma regra para todos os grafos, em todos os passos.
  • A política pode misturar estratégias ao longo da trajetória.
  • Exemplo: começar pelos hubs (como Degree) e terminar pela conectividade (como BFS).
Evidência: a entropia da política decresce ao longo do treino (Figura 7.1) — de exploração para decisões cada vez mais confiantes. A política aprende a se concentrar.
Bloco 4 · Resultados

Protocolo experimental (Barabási–Albert)

Dataset:

  • 6000 grafos BA
  • \(m_{BA} = 2\)
  • \(N \in [20, 60]\)

Métricas:

  • valid-size ratio: fração com tamanho no intervalo correto
  • connected ratio: fração de grafos conexos
  • avg size: tamanho médio gerado
  • Gini: desigualdade da distribuição de graus
  • hubiness: concentração de grau nos nós de maior grau

GCN RL vs. Attention RL: a progressão

GCN RL (primeira versão)
  • valid-size = 1.0000
  • connected = 0.6334

Prova a viabilidade de aprender a ordenação, mas falha em conectividade.

Attention RL (versão final)
  • valid-size = 1.0000
  • connected = 0.9960

Atenção + 3 camadas + warm start resolvem o gap de conectividade.

As mudanças arquiteturais são motivadas pela análise do modo de falha do GCN RL.

Método valid-size connected avg size hubiness

Evolução da loss nas iterações externas

  • Comportamento oscilatório — esperado em otimização alternada com RL.
  • Melhor checkpoint na iteração 61 — a política continua explorando ao invés de colapsar.
  • Evidência de que o sistema não converge prematuramente.

Interpretação dos resultados

  • A política não copia a ordenação por grau — ela vai além do warm start.
  • O RL refina o prior hub-aware para ganhar conectividade — estratégia híbrida genuína.
  • Evidência: a entropia decresce, mas não colapsa — a política mantém diversidade onde necessário.
A ordenação aprendida combina intuições de múltiplas heurísticas de forma adaptativa, grafo a grafo.
Bloco 5 · Conclusão

Contribuições da dissertação

  • Análise empírica controlada do efeito de ordenações determinísticas no DGMG, mostrando que cada topologia favorece uma heurística diferente.
  • Formulação do MDP de ordenação com recompensa composta (preditiva + estrutural), permitindo tratar a serialização como problema de otimização.
  • Attention RL Ordering: método que atinge resultado forte em Barabási–Albert, sendo o único a equilibrar simultaneamente tamanho, conectividade e estrutura de hubs.

Limitações e trabalhos futuros

Limitações

  • Avaliação de RL concentrada em grafos BA sintéticos
  • Custo computacional maior que heurísticas
  • Alta variância do estimador REINFORCE

Trabalhos futuros

  • Datasets reais: moléculas, redes biológicas
  • Redução de variância: actor-critic, importance sampling
  • Análise interpretativa do que a política aprendeu
  • Extensão para grafos com atributos e tipos de arestas

Mensagem final

"A ordenação dos nós não é pré-processamento neutro — é parte do modelo.
Tratá-la como alvo de otimização melhora a geração autorregressiva de grafos."
